# 🏗 Design Pattern

## 문제 해결 능력의 향상

소프트웨어 설계에서 자주 맞닥뜨리는 문제가 있고, 많은 이들이 이를 해결하기 위한 효과적인 해결 방법을 강구했습니다. 자신이 접한 문제는 대부분 다른 이들이 먼저 접했습니다. 선배 개발자의 노고가 담긴 [소프트웨어 설계 노하우](#user-content-fn-1)[^1]를 공부하면서 시행착오를 줄이고, 나아가 왜 이런 해결 방법을 제시했는지 근본적인 물음에 다가갈 수 있습니다.



## 협업 스킬의 향상

해결 방법을 정제된 용어로 정의했기 때문에, 동료와의 소통을 효율적으로 진행할 수 있습니다.

> 메인 비즈니스 로직을 만들고 세부 기능은 서브 클래스에게 위임해서,\
> 큰 흐름은 같되 자잘한 내용은 구현체에 따라 다르게 설계해봅시다.

이렇게 말하는 것과

> 템플릿 메서드 패턴으로 설계해봅시다.

라고 말하는 것중에, 어느 것이 이해하기 쉽고 오해의 여지가 적은지는 누구나 알 수 있을 것입니다.
